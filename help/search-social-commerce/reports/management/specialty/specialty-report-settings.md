# 특성 보고서 설정

| 탭 | 매개 변수 | 설명 |
|----|----|----|
| 해당 사항 없음 | [!UICONTROL Name] | (선택 사항) 보고서 및 템플릿 이름(보고서를 템플릿으로 저장하는 경우). 기존 템플릿을 적용하는 경우 기본적으로 템플릿 이름이 채워집니다. 템플릿을 적용하지 않거나 이름을 입력하지 않으면 보고서 이름이 지정됩니다 `<client name>-<date and time>-<report type>` (예: &quot;acme - 2009년 4월 3일 11:25:오전 19시 PDT - Keyword&quot;)로 설정됩니다.<br><br>선택적으로 사용자 지정 이름을 입력할 수 있지만 파일 확장명은 사용하지 마십시오.<br><br>템플릿을 만드는 경우 [ftp 디렉터리에 보고서 보내기](/help/search-social-commerce/reports/automation/ftp-reports.md)그런 다음 선택적으로 파일 이름의 아무 곳에나 &quot;CSV&quot;(대문자)를 포함하여 기본 TSV 형식이 아닌 CSV 형식으로 파일을 만들 수 있습니다. 다음을 참조하십시오 [ftp 디렉토리로 전송된 보고서의 파일 이름 요구 사항](/help/search-social-commerce/reports/automation/ftp-reports.md). |
|  | [!UICONTROL Save as template] | (일정에 따라 보고서를 실행하지 않으려는 경우 선택 사항) 보고서 설정을 템플릿으로 저장합니다. 템플릿은에서 사용할 수 있습니다. [!UICONTROL Reports] > [!UICONTROL Report Templates] 를 보고 새 보고서를 만드는 데 재사용할 수 있습니다. 보고서를 템플릿으로 저장하려면 확인란을 선택합니다.<br><br>일정에 따라 보고서를 실행하려면 설정을 템플릿으로 저장해야 합니다.<br><br><b>참고:</b> 기존 템플릿을 기반으로 하는 경우에도 현재 매개변수 세트를 새 템플릿으로 저장할 수 있습니다. |
|  | [!UICONTROL Type] | 생성할 보고서 유형입니다. |
| [!UICONTROL Basic  Settings] | [!UICONTROL Template] | (선택 사항) 적용할 보고서 템플릿으로, 템플릿에 따라 보고서 옵션을 미리 채웁니다. 보고서 유형에 대해 저장되고 사용할 수 있는 모든 템플릿이 나열됩니다.<br><br>템플릿을 선택하면 보고서 옵션을 계속 변경할 수 있으며 보고서를 새 템플릿으로 저장할 수도 있습니다. |
|  | [!UICONTROL Data Aggregation] | ([!UICONTROL Forecast Accuracy Report only]) 보고서의 각 행에 표시되어야 하는 시간 단위: <i>[!UICONTROL Summary (the default)]</i>, <i>[!UICONTROL Daily]</i>, <i>[!UICONTROL Weekly]</i> (그리고 주의 첫 번째 요일을 지정합니다), <i>[!UICONTROL Monthly]</i>, <i>[!UICONTROL Quarterly]</i>, <i>[!UICONTROL Yearly]</i>, 또는 <i>[!UICONTROL Day of Week]</i> (기본 보고서만).<br><br>이 보고서는 지정된 날짜 범위의 각 시간 단위에 대해 각 키워드 또는 다른 항목에 대해 하나의 행을 표시합니다. 예를 들어, <i>[!UICONTROL Summary]</i> 및 날짜 범위 <i>[!UICONTROL Last 7 Days]</i> 키워드 보고서의 경우 보고서에는 각 키워드에 대해 한 개의 행이 포함되며, 각 행에는 7일 동안의 합계 및 평균이 요약됩니다. 과 동일한 보고서 <i>[!UICONTROL Daily]</i> 데이터 집계에는 지난 7일 동안 각 키워드에 대해 한 개의 행이 포함됩니다(키워드당 7개의 행).<br><br><b>참고:</b><ul><li>시간별로 데이터를 집계할 때 &quot;[!UICONTROL Hourly]&quot; 또는 &quot;[!UICONTROL Day of Week (Hourly)]&quot; 집계:</li><ul><li>최대 14일의 데이터를 볼 수 있습니다. 보고서는 24시간 시계에 시간별로 데이터를 표시합니다. &quot;[!UICONTROL 0]&quot;00 포함:01-00:59 (12):01-12:오전 59시) 및 &quot;[!UICONTROL 23]&quot;에 23개 포함:01-23:59 (11):01-11:광고주의 시간대인 오후 59시).</li><li>&quot;[!UICONTROL Device]&quot;열은 지원되지 않습니다.</li><li>의 경우 [!UICONTROL Ad Group Report], &quot;[!UICONTROL Data Based On]&quot; 옵션 &quot;[!UICONTROL Portfolio Configuration During the Specified Dates]&quot;은(는) 지원되지 않습니다.</li><li>의 경우 [!UICONTROL Ad Group Report], 현재 포트폴리오 구성을 기준으로만 시간별 데이터를 사용할 수 있습니다(&quot;[!UICONTROL Data Based On]&quot; 옵션 &quot;[!UICONTROL Current Portfolio Configuration]&quot;), 지정된 날짜 동안 포트폴리오 구성이 아닙니다.</li></ul></ul> |
|  | [!UICONTROL Conversions Based on] | ([!UICONTROL AdWords Shopping Performance Report], [!UICONTROL Campaign Daily Impression Share Report], 및 [!UICONTROL Keyword Daily Impression Share Report] 전용) 전환 데이터를 보고하는 방법:<ul><li><i>[!UICONTROL Transaction date]</i> (기본값): 트랜잭션 일자가 지정된 기간 동안 발생한 트랜잭션을 확인합니다. 이 옵션은 지정된 기간 내에 얼마나 많은 수익을 얻었는지 보여 줍니다.</li><li><i>[!UICONTROL Click date]:</i> 지정된 기간 동안 발생한 클릭으로 인한 트랜잭션을 보려면 다음을 수행합니다. 포트폴리오에 클릭과 거래 사이에 상당한 지연이 있는 경우 이 옵션은 포트폴리오에 대한 클릭당 기록 매출을 계산하는 데 유용하며 이는 시간이 지남에 따라 예상되는 매출 동작을 나타냅니다.</li></ul> |
|  | [!UICONTROL Date Range] | 데이터를 생성할 날짜 범위:<ul><li><i>[범위 사전 설정]:</i> 다음 범위의 공통 시간 증분 목록: <i>[!UICONTROL Today]</i> 끝 <i>[!UICONTROL Last 180 Days]</i>. 기본값은 입니다 <i>[!UICONTROL Last 7 Days]</i>: 데이터를 사용할 수 있는 지난 7일을 보고합니다. <b>참고:</b> <i>[!UICONTROL Last Month]</i>, <i>[!UICONTROL Last 3 Months]</i>, 및 <i>[!UICONTROL Last 6 Months]</i> 이전 달력의 데이터를 표시합니다.</li><li><i>[!UICONTROL Custom Date Range]:</i> 시작 날짜와 종료 날짜를 지정합니다. 날짜를 MM/DD/YYYY 또는 M/D/YYYY 형식으로 입력하거나 ![캘린더](/help/search-social-commerce/assets/calendar.png "캘린더") 필드 옆에 있는 날짜를 선택합니다.</li></ul> |
|  | [!UICONTROL Compare With] | 지정된 날짜 범위의 데이터를 두 번째 날짜 범위의 데이터와 비교합니다. 이 옵션을 선택하면 각 일반 데이터 열에 대해 두 개의 열이 더 추가됩니다. 예를 들어 &quot;노출 횟수&quot;에 대해 하나의 열만 포함하는 대신 보고서에는 &quot;에 대한 열이 포함됩니다[!UICONTROL Impressions Range 1],&quot; &quot;[!UICONTROL Impressions Range 2]및 &quot;[!UICONTROL Impressions Difference].&quot;<br><br><b>참고:</b><ul><li>사용자 지정/파생 지표에 대해서는 차이점 열이 표시되지 않습니다.</li><li>큰 날짜 범위를 비교하는 보고서를 생성하는 데 시간이 더 오래 걸립니다.</li></ul> |
|  | [!UICONTROL Show Comparison Data as] | (다음과 같은 경우)[!UICONTROL Compare With]&quot;선택됨) &quot;&quot;에서 선택한 두 날짜 범위의 데이터 차이를 표시하는 방법[데이터 필드] [!UICONTROL Difference]&quot; 열:<ul><li><i>[!UICONTROL Variance]</i> (기본값): 차이를 숫자 값으로 표시합니다.</li><li><i>[!UICONTROL % Change]:</i> 차이를 백분율로 표시합니다.</li></ul> |
|  | [!UICONTROL Filter By] | ([!UICONTROL RSA Asset Report], [!UICONTROL Campaign Daily Impression Share Report], 및 [!UICONTROL Keyword Daily Impression Share Report] 전용) 특정 포트폴리오에 대한 데이터 또는 특정 광고 네트워크에 대한 데이터에 대해 보고할지 여부입니다.<ul><li><i>[!UICONTROL Portfolio]</i> (기본값): 하나 이상의 포트폴리오에 캠페인용 데이터를 포함합니다.</li><li><i>[!UICONTROL Search Engine]:</i> 보고서 유형에 적용할 수 있는 경우 하나 이상의 광고 네트워크에 캠페인용 데이터를 포함합니다. |
|  | \[기본 필터\] | 포함할 데이터 구성 요소입니다. 선택하지 않으면 모든 포트폴리오(해당되는 경우) 또는 모든 적용 가능한 광고 네트워크 및 해당 하위 구성 요소에 대한 데이터가 보고서에 포함됩니다. 필요한 경우 개별 구성 요소 및 하위 구성 요소를 지정하여 보고할 데이터의 범위를 좁힐 수 있습니다. 보고서 유형 및 (해당되는 경우) 포트폴리오별 또는 광고 네트워크별로 필터링하는지 여부에 따라 포함할 구성 요소를 지정합니다.<ul><li><i>[!UICONTROL Portfolio]:</i> 하나 이상의 포트폴리오 또는 해당 하위 구성 요소(캠페인 또는 광고 그룹). 구성 요소와 모든 하위 구성 요소를 선택하려면 구성 요소 이름 옆에 있는 확인란을 선택합니다. 하위 구성 요소를 선택하려면 하위 구성 요소 이름 옆의 확인란을 선택한 다음 >> 를 클릭하여 하위 구성 요소를 [!UICONTROL Selected Filters] 열. 예를 들어, Portfolio 1과 모든 해당 캠페인 및 광고 그룹에 대한 데이터를 가져오려면 Portfolio 1 옆에 있는 확인란을 선택합니다.
